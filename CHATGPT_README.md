# ChatGPT 项目快速总览

## 当前最高目标

本项目的最高目标不是搭建知识库本身，也不是单纯批量生成内容。

当前最高目标是：

- 持续分析小红书店铺数据
- 持续产出能提升流量、点击、进店、订单的爆款笔记
- 所有系统建设都必须服务于订单越来越多、数据越来越好

判断系统是否有效，只看内容数据和成交数据是否持续变好。

## 当前主推产品

- 产品名称：班级通知
- 售价：17.9 元
- 成交情况：小红书店铺已成交 400 多单
- 目标用户：老师，尤其是班主任、任课老师、办公室老师
- 核心痛点：老师不用跑教室叫学生、叫班长、叫课代表

产品真实使用方式：

老师在办公室电脑或手机发送通知，教室电脑或一体机接收端直接弹出通知，学生抬头就能看到。

## 当前发布包规则

每套发布包必须包含：

- 主图.png
- 副图1.png
- 副图2.png
- 副图3.png
- 副图4.png
- 副图5.png
- 发布文案.md
- info.json
- 做图交接单.md
- 状态.md

`info.json` 的 `content` 必须使用 `\n\n` 保留段落空行。

发布助手优先读取 `info.json`，不要依赖 Markdown 浏览器预览里的复制效果。

## 班级通知默认做图流程

默认不再拆成“先做主图、主图复核、再做副图”。

普通 6 篇测试、日常批量发布、快速测数据时，每套笔记默认一次性生成完整图片包：

- 主图.png
- 副图1.png
- 副图2.png
- 副图3.png
- 副图4.png
- 副图5.png

标准流程：

1. 生成 6 套笔记方案。
2. 为每套创建发布包文件夹。
3. 每套生成完整做图提示词。
4. 做图窗口一次性生成主图 + 副图1-副图5。
5. 图片保存到对应发布包文件夹。
6. 检查发布包完整性。
7. 状态改为“待发布”。
8. 发布助手读取发布包。
9. 发布后采集数据。
10. 数据复盘并更新知识库。

主图质检只是可选流程，只在用户明确要求“先看主图”、主图明显跑偏、文字错误或需要打磨爆款主图模板时触发。

默认完整做图流程不需要把图片同步 GitHub 给 ChatGPT 看。

效率优先于精修。先发出去测数据，再根据真实阅读、点击、加购、成交结果决定是否优化图片和文案。
