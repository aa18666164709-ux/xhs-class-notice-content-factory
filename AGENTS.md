# AI 协作规则

本项目是“班级通知小红书内容增长系统”，不是单纯的知识库模板，也不是批量灌内容工具。

AI 执行任何任务时，必须优先服务于一个目标：

提高班级通知产品在小红书的流量、点击、进店、成交和复购可能性。

## 每次任务前必须先读

1. `CLAUDE.md`
2. `PROJECT_STATUS.md`
3. `00-系统协议与总规则/当前主推产品说明.md`
4. `02-产品资产库/01-教师产品/班级通知.md`
5. `09-创作规则库/班级通知内容生产总规则.md`
6. `14-人工修改学习库/写作偏好档案.md`
7. `16-复盘进化库/系统进化规则.md`

## 内容判断原则

- 数据优先：有店铺数据时，优先根据数据决策。
- 订单优先：高播放但无点击、无订单的方向不能盲目复刻。
- 转化优先：低播放但有点击、有订单的方向要继续优化。
- 场景优先：班级通知必须围绕老师真实场景。
- 复盘优先：发布后必须回看数据，把有效规律写回知识库。
- 简化优先：新功能必须服务于数据提升和订单增长。

## 禁止写偏

班级通知不能写成：

- 家长通知
- 回执统计
- 家校沟通
- 复杂管理系统
- 泛泛的软件神器
- 全校数字化平台

## 发布包规则

发布包图片本体保存在本地 E 盘成品目录，不复制进 GitHub。

发布包索引、发布文案结构、info.json 规则可以进入 GitHub。

每个发布包必须包含：

- 主图.png
- 副图1.png
- 副图2.png
- 副图3.png
- 副图4.png
- 副图5.png
- 发布文案.md
- info.json

info.json 的 content 字段必须使用 `\n\n` 保留真实段落换行。

## GitHub 同步边界

公开仓库不允许上传：

- Cookie
- Token
- API Key
- 账号密码
- 客户隐私
- 完整订单明细
- 激活卡号完整列表
- network-response 原始文件
- HAR 文件
- E 盘图片原图
