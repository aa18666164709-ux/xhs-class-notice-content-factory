# 成品图片输出与发布助手读取规范

## 固定成品图片输出根目录

E:\优逸店资料截图\班级通知\笔记素材\生成成品图片

以后所有班级通知小红书成品图片，都必须整理到这个固定目录下。

## 一套笔记一个文件夹

每一套笔记必须单独新建文件夹，不要把多套笔记图片混放在同一个目录。

## 文件夹命名格式

YYYYMMDD_场景关键词_版本说明

示例：

- 20260517_班长拿试卷_全新生成_2
- 20260517_临时叫学生_复刻高成交款
- 20260517_课代表拿作业_场景轮换款

命名要求：

- 日期使用 8 位数字。
- 场景关键词必须具体，例如“班长拿试卷”“临时叫学生”“课代表拿作业”。
- 版本说明用于区分复刻高成交款、场景轮换款、自由测试款、全新生成等。
- 不要只写“新图”“最终版”“测试图”。

## 发布文案.md 固定格式

发布文案.md 必须使用以下固定格式，方便人工检查。发布插件优先读取 info.json，不再依赖浏览器里的 Markdown 预览文本。

```markdown
# 方案：xxx

## 正文标题
这里写小红书标题

## 正文内容
这里写正文，保留换行格式

## 标签
#标签1 #标签2 #标签3

## 首评
这里写第一条评论

## 验收重点
这里写发布前需要检查的重点
```

## 做图 Skill 完成后的整理规则

以后做图 Skill 生成完图片后，如果图片散落在其他地方，必须整理到固定目录：

E:\优逸店资料截图\班级通知\笔记素材\生成成品图片

整理时必须完成：

1. 为每套笔记新建独立文件夹。
2. 将主图改名为“主图.png”。
3. 将副图按发布顺序改名为“副图1.png”“副图2.png”“副图3.png”“副图4.png”“副图5.png”。
4. 如有更多副图，继续命名为“副图6.png”。
5. 生成或补齐“发布文案.md”。
6. 生成或补齐“info.json”。
7. 保证发布文案.md 中包含标题、正文、标签、首评、验收重点。
8. 保证 info.json 中包含 title、content、tags、comment、product、scheduleAt。

## info.json 固定格式

发布插件优先读取 info.json。

```json
{
  "title": "笔记标题",
  "content": "第一段正文。\n\n第二段正文。\n\n第三段正文。",
  "tags": ["班主任", "老师日常", "教师工具"],
  "comment": "这里是首评",
  "product": {
    "id": "",
    "name": "班级通知"
  },
  "scheduleAt": ""
}
```

要求：

- content 字段必须保留真实段落换行 `\n\n`。
- tags 必须是数组，不带 `#`。
- product.name 默认写“班级通知”。
- product.id 和 scheduleAt 未指定时留空。
- 如果 content 不包含 `\n\n`，发布包不能进入发布队列。

## 发布助手读取范围

发布助手只读取以下路径下面的每个子文件夹：

E:\优逸店资料截图\班级通知\笔记素材\生成成品图片

发布助手不应该读取知识库里的草稿文件夹，也不应该读取散落在其他目录里的图片。

## 发布助手读取逻辑建议

1. 读取固定根目录下最新日期的文件夹。
2. 检查是否存在“主图.png”。
3. 按顺序读取“副图1.png”“副图2.png”“副图3.png”“副图4.png”“副图5.png”。
4. 如果存在“副图6.png”，按顺序追加。
5. 读取“info.json”。
6. 从 info.json 中提取 title、content、tags、comment。
7. 上传图片时保持“主图.png”为第一张。
8. 正文必须使用 info.json.content 中的真实换行。
9. 标签按 info.json.tags 数组处理。
10. 首评单独作为第一条评论处理。
11. 发布文案.md 仅作为人工检查和兜底备份。

## 不进入发布队列的情况

如果某个文件夹缺少以下任一文件，则不进入发布队列：

- 主图.png
- 发布文案.md
- info.json

如果缺少副图1.png 到 副图5.png，应进入“待补图/待检查”状态，不建议直接发布。

## 发布前人工检查

发布前必须人工确认：

- 主图是否为第一张。
- 主图中文字是否能在手机缩略图中看清。
- 副图顺序是否正确。
- 正文换行是否保留。
- 标签是否准确。
- 首评是否没有写成硬广。
- 是否没有写成家长通知、回执统计、家校沟通。
- 是否没有暴露不该发布的内部路径或文件名。
