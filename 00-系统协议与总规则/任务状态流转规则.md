# 任务状态流转规则

## 总流转链路

输入素材 -> 爆款案例 -> 爆款拆解 -> 爆款迁移 -> 笔记生成 -> 做图交接 -> 做图成品 -> 人工发布 -> 发布数据回流 -> 复盘进化

## 1. 输入素材

位置：

- `01-输入素材池/`

可放入：

- 用户评论截图
- 私信截图
- 同行店铺截图
- 自己店铺数据截图
- 临时灵感
- 待清洗文本

进入下一步条件：

- 素材能对应到班级通知真实场景。
- 素材能说明老师的具体麻烦。
- 素材不偏向家长通知、回执统计或家校沟通。

## 2. 爆款案例

位置：

- `04-爆款案例库/`

状态：

- 待拆解
- 同行业爆款
- 跨行业爆款
- 高价值收藏
- 不适合迁移

进入拆解条件：

- 有明确标题、主图、正文或互动数据。
- 能看出爆点来自痛点、画面、标题、情绪或转化。
- 可迁移到班级通知场景。

## 3. 爆款拆解

位置：

- `05-爆款拆解库/`

必须拆出：

- 标题公式
- 主图钩子
- 痛点场景
- 图片顺序
- 正文结构
- 转化动作
- 可迁移点
- 不可照搬点

进入迁移条件：

- 明确迁移的是结构，不是照抄原文。
- 已判断适合班级通知哪个具体场景。

## 4. 爆款迁移

位置：

- `06-爆款迁移库/`

迁移对象：

- 老师不用跑教室叫学生
- 班长拿试卷
- 课代表拿作业
- 组长收作业本
- 临时通知班级
- 教室一体机弹出通知

输出要求：

- 新标题方向
- 新主图文案
- 新电子屏幕文案
- P1-P6 结构
- 正文转化方式

进入生成条件：

- 场景足够具体。
- 产品承接自然。
- 没有写偏成家长通知。

## 5. 笔记生成

位置：

- `11-笔记生成任务/`
- `12-笔记生成结果/01-待检查/`

生成内容必须包含：

- 小红书标题
- 正文
- 标签
- 首评
- P1-P6 图片方案
- 做图交接单
- AI 自检结果

状态流转：

- 待生成 -> 已生成 -> 待检查 -> 待修改 / 待发布

## 6. 做图交接

位置：

- `12-笔记生成结果/`

做图交接单必须包含：

- 产品信息
- 本篇笔记主题
- 图片数量与尺寸
- 整体风格
- 素材说明
- P1-P6 每张图内容
- 每张图电子屏幕文案
- 发布文案
- 特别要求

进入做图条件：

- 人工确认文案可用。
- P1 主图明确。
- 电子屏幕文案短、大、清楚。
- 没有产品方向错误。

## 7. 做图成品

由做图 Skill 窗口完成。

人工必须检查：

- 图片是否为 3:4。
- 文字是否正确。
- 电子屏幕文案是否清楚。
- 是否像老师真实场景。
- 是否避免过度 PPT 感或广告感。

合格后进入待发布。

## 8. 发布

位置：

- `12-笔记生成结果/03-待发布/`
- `12-笔记生成结果/04-已发布/`
- `13-内容流水线/05-已发布.md`

发布必须人工执行。

发布后记录：

- 发布时间
- 标题最终版
- 正文最终版
- 图片最终版
- 链接
- 商品挂载情况

## 9. 发布数据回流

位置：

- `15-发布数据与店铺数据/`

记录数据：

- 阅读/播放
- 商品点击
- 商品点击率
- 订单数
- 成交金额
- 支付转化率
- 评论
- 私信

低于预期的进入低效复盘，高于预期的进入高效复盘。

## 10. 复盘进化

位置：

- `16-复盘进化库/`
- `14-人工修改学习库/`

沉淀内容：

- 高转化场景
- 高点击标题
- 高点击主图结构
- 高转化电子屏幕文案
- 低效表达
- 禁用词
- 人工修改偏好

## 关键状态判断

- `待检查`：AI 已生成，但未人工判断。
- `待修改`：方向可用，但标题、正文或图片方案需改。
- `待发布`：人工确认可发布。
- `已发布`：已人工发布到小红书。
- `已复盘`：已录入数据并沉淀结论。

## 不允许跳过

- 不允许未自检直接做图。
- 不允许未人工检查直接发布。
- 不允许无数据就写入高效规则。
- 不允许把低效内容继续复刻。
