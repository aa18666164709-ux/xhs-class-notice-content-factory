# GitHub 临时图片同步与清理规则

## 核心原则

GitHub 仓库只保存系统规则、Skill、发布包索引、数据复盘摘要和运行日志摘要。

GitHub 不长期保存完整图片素材。

## 允许临时同步的图片

只允许同步以下图片：

1. 6张主图拼图预览图
2. 必须让 ChatGPT 复核的单张问题主图

不允许同步：

1. 副图
2. 完整发布包图片
3. 所有主图原图批量集合
4. E 盘成品图片原图
5. 做图过程中的临时图

## 临时图片固定目录

所有给 ChatGPT 临时查看的图片，统一放到：

`90-临时给ChatGPT看的图片/`

文件命名规则：

```text
latest_6张主图拼图预览.jpg
latest_主图质检报告.md
```

如果有单张问题图，可以临时放：

```text
problem_01.jpg
problem_02.jpg
```

## 覆盖规则

每次新一轮 6 套图生成后，只保留最新一轮预览图。

新的 `latest_6张主图拼图预览.jpg` 可以覆盖旧文件。

不要为每一轮都长期新增图片文件。

## 主图归档上传流程

用户说“运行主图归档并上传GitHub”时，执行以下流程：

1. 扫描本轮 6 个发布包文件夹。
2. 确认每个发布包都有 `主图.png`。
3. 如果 6 张主图不齐，停止并列出缺失项。
4. 如果 6 张主图齐全，生成 6 张主图拼图预览。
5. 将压缩后的拼图保存为：`90-临时给ChatGPT看的图片/latest_6张主图拼图预览.jpg`
6. 提交并推送到 GitHub。
7. 用户只让 ChatGPT 复核主图。

本流程不上传副图，不上传 6 张主图原图，不上传完整发布包图片。

## 清理规则

ChatGPT 复核完成后，必须执行清理：

1. 删除 `90-临时给ChatGPT看的图片/` 下的 jpg/png/webp 图片。
2. 保留或转移文字质检结论到：`16-复盘进化库/主图复核结论/`
3. GitHub 上不长期保留图片。
4. 删除图片后提交并推送。

## 历史结论保留

图片删除后，只保留文字记录：

- 本轮主题
- 哪些主图通过
- 哪些主图需要重做
- 通过原因
- 重做原因
- 下一轮主图优化建议

保存到：

`16-复盘进化库/主图复核结论/YYYYMMDD_主图复核结论.md`

## 禁止事项

- 不要把 E 盘完整发布包图片同步到 GitHub。
- 不要上传副图。
- 不要把图片长期留在 GitHub。
- 不要让 GitHub 仓库变成图片素材库。
- 不要把客户信息、后台截图、订单隐私图片上传 GitHub。
