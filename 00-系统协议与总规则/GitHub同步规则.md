# GitHub 同步规则

## 仓库定位

公开 GitHub 仓库只同步“班级通知小红书内容增长系统”的公开项目状态。

允许同步：

- 系统规则
- 内容模板
- Skill 说明文档
- 笔记方案
- 做图交接单
- 发布包索引
- 数据复盘摘要
- 运行日志摘要

## 禁止同步内容

禁止上传：

- Cookie
- Token
- API Key
- 账号密码
- 客户隐私
- 完整订单明细
- 激活卡号完整列表
- 带登录态的 network-response 原始文件
- HAR 文件
- 浏览器缓存
- 原始接口响应
- E 盘成品图片原图，除非用户明确要求

## Push 前检查

每次 push 前必须先执行敏感信息检查。

每次 push 前必须先执行：

```powershell
git status
```

如果发现敏感文件，必须停止 push，输出风险路径和原因，等待人工确认。

## .gitignore 要求

`.gitignore` 必须持续排除：

- 环境变量文件
- Cookie / Token / Secret / Credentials
- network-response / HAR
- 原始接口响应
- 客户隐私
- 订单明细
- 激活卡号完整列表
- 缓存图片
- 依赖目录
- 原始日志

## 同步原则

进入公开仓库的内容必须默认可公开。

不要为了方便让 GitHub 承担私密数据存储职责。

如果某个文件既包含公开规则又夹带私密数据，必须先拆分：公开部分进入仓库，私密部分留在本地并加入 `.gitignore`。
