# 主图质检规则

## 规则定位

本规则用于“班级通知”小红书图文笔记的轻量版图片质检流程。

主图决定点击率，所以每轮只深度复核主图。副图不做视觉质检，只检查发布包文件是否存在、命名是否正确。

## 质检对象

固定成品图片输出根目录：

`E:\优逸店资料截图\班级通知\笔记素材\生成成品图片`

每轮默认质检 6 套发布包，对应 6 张 `主图.png`。

如果本轮发布包数量不是 6 套，必须在报告里写明实际数量，不要凭空补齐。

## 主图归档并上传 GitHub 标准流程

本流程用于用户说“运行主图归档并上传GitHub”时执行。

标准流程：

1. 6 个做图窗口各做 1 张主图。
2. 主图保存到各自发布包文件夹，文件名必须是 `主图.png`。
3. Codex 扫描本轮 6 个发布包文件夹。
4. Codex 确认 6 张 `主图.png` 都齐了。
5. 如果主图缺失，停止并列出缺失文件夹。
6. 如果主图齐全，生成 6 张主图拼图预览。
7. 将压缩后的拼图同步到 GitHub 临时目录。
8. 用户让 ChatGPT 看 GitHub 或让 Codex 把拼图发给用户。
9. 用户和 ChatGPT 只复核主图。

GitHub 临时目录：

`90-临时给ChatGPT看的图片/`

固定文件名：

`latest_6张主图拼图预览.jpg`

本流程不上传 6 张主图原图，不上传副图。

## 必读文件

每个发布包只重点读取：

- `主图.png`
- `发布文案.md`
- `info.json`

副图只检查是否存在：

- `副图1.png`
- `副图2.png`
- `副图3.png`
- `副图4.png`
- `副图5.png`

## 副图处理原则

副图不做视觉质量分析。

副图不判断：

- 好不好看
- 排版是否高级
- 字体是否更优
- 画面是否更有设计感

副图只判断：

- 文件是否存在
- 命名是否正确
- 是否缺少发布助手要求的固定文件

只要副图存在、命名正确，就默认通过。

## 主图检查标准

每套只深度检查 `主图.png`。

主图必须优先判断：

1. 主标题是否足够大
2. 主标题是否是具体老师场景
3. 是否突出“老师不用跑教室叫人”
4. 是否出现办公室老师 + 教室一体机/大屏
5. 屏幕文案是否大、短、清楚
6. 是否跑偏成家长通知、家校沟通、回执统计
7. 是否像复杂软件后台
8. 是否广告感过重
9. 是否值得发给 ChatGPT 做最终判断

## 通过标准

主图可以判定为“通过”的条件：

- 手机缩略图里能看清主标题
- 主标题是老师真实痛点，不是抽象产品名
- 能看懂“办公室发送 -> 教室屏幕弹出”的使用链路
- 屏幕文案短、大、清楚
- 没有写偏成家长通知、回执统计或家校沟通
- 不像复杂后台或硬广海报

## 需 ChatGPT 复核标准

出现以下情况时，标记为“需 ChatGPT 复核”：

- 主标题方向可能对，但视觉点击力不确定
- 屏幕文案可读性不确定
- 场景能看懂，但使用链路不够一眼明白
- 广告感略重，但未严重跑偏
- 需要在 6 张主图里横向比较哪张更值得先发

## 建议重做标准

出现以下情况时，标记为“建议重做”：

- 主标题太小
- 主标题偏产品名、软件名、系统名
- 看不出老师不用跑教室叫人的痛点
- 屏幕文案太长或看不清
- 画面像家长通知、家校沟通或回执统计
- 画面像复杂软件后台
- 广告感过重，缺少老师真实场景

## 主图拼图要求

每轮把 6 张 `主图.png` 拼成一张总览图。

拼图要求：

- 只拼主图
- 不拼副图
- 每张主图下面标注主题或文件夹名
- 文件名：`YYYYMMDD_班级通知_6张主图拼图预览.png`
- 保存到：`18-运行日志/主图质检/`
- 另存压缩 JPG 到：`90-临时给ChatGPT看的图片/latest_6张主图拼图预览.jpg`

## 主图质检报告

报告路径：

`18-运行日志/主图质检/YYYYMMDD_班级通知_主图质检报告.md`

报告必须包含：

- 检查时间
- 发布包数量
- 主图数量
- 副图是否齐全
- 6 张主图拼图本地路径
- 6 张主图拼图 GitHub 路径
- 每张主图的初步判断
- 缺失或命名错误的副图
- 需要发给 ChatGPT 复核的主图路径
- 建议重做的主图路径

## GitHub 同步边界

只同步：

- 压缩后的主图拼图预览 JPG
- 主图质检报告

不要同步：

- 全部副图
- 6 张主图原图
- 完整图片库
- E 盘成品图片原图全集

## 输出要求

完成一次主图质检后，只输出：

- 主图拼图本地路径
- 主图拼图 GitHub 路径
- 主图质检报告路径
- 哪些主图通过
- 哪些主图需要 ChatGPT 复核
- 哪些主图建议重做
