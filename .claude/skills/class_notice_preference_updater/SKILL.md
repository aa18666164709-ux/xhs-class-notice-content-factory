# class_notice_preference_updater

## 功能

对比 AI 原稿和人工修改稿，提炼用户的写作偏好、标题偏好、主图偏好、禁用表达和结构偏好。

本 Skill 只总结稳定规律，不生成新笔记，不覆盖旧偏好。

## 触发语

- 更新偏好
- 学习我的修改
- 分析人工改稿
- 更新班级通知写作偏好

## 输入来源

优先读取：

- `12-笔记生成结果/01-待检查`
- `12-笔记生成结果/02-人工修改`
- `14-人工修改学习库/写作偏好档案.md`
- `09-创作规则库/禁用表达.md`

如果 `09-创作规则库/禁用表达.md` 不存在，可读取：

- `09-创作规则库/08-禁用词与低效表达.md`
- `09-创作规则库/班级通知内容生产总规则.md`

## 输出位置

- `14-人工修改学习库/写作偏好档案.md`
- `16-复盘进化库/人工修改规律.md`
- `09-创作规则库/禁用表达.md`

如果输出文件不存在，先创建。

## 分析对象

对比以下内容：

- 标题原稿 vs 人工标题
- P1 主图文案原稿 vs 人工主图文案
- 电子屏幕文案原稿 vs 人工屏幕文案
- 正文开头原稿 vs 人工正文开头
- 标签原稿 vs 人工标签
- 首评原稿 vs 人工首评
- 做图交接单原稿 vs 人工修改版

## 只总结稳定规律

不要记录偶然的一次性修改。只有符合以下条件之一，才写入偏好：

- 同类修改出现 2 次以上。
- 用户明确说“以后都这样”。
- 修改明显符合已有偏好，例如更具体、更少广告感、更像老师分享。
- 修改能降低写偏风险，例如删掉家长通知、回执统计、复杂系统。

## 不覆盖旧偏好

更新方式：

- 只追加新发现。
- 保留已有偏好。
- 如果新偏好与旧偏好冲突，写入“待确认区”。
- 不直接删除旧规则，除非用户明确要求。

## 必须提炼的维度

### 写作偏好

- 先痛点后产品
- 具体场景优先
- 老师真实分享
- 少广告感

### 标题偏好

- 主图短狠具体
- 优先写“不用跑教室叫人”
- 优先写“班长拿试卷 / 课代表拿作业 / 叫学生来办公室”

### 主图偏好

- 电子屏幕文案大、短、清楚
- 产品名不要做最大字
- 画面像真实办公室和教室

### 禁用表达

- 家长通知
- 回执统计
- 家校沟通
- 复杂管理系统
- 泛泛软件神器
- 全校必备

### 结构偏好

- P1 强痛点
- P2 真实场景
- P3 旧方法麻烦
- P4 新方法
- P5 屏幕弹出效果
- P6 适用场景和交付方式

## 待确认区

如果发现冲突，写入：

```markdown
## 待确认偏好

- 冲突点：
- 旧规则：
- 新修改：
- 建议确认问题：
```

## 输出要求

每次更新后，必须说明：

- 新增了哪些偏好
- 新增了哪些禁用表达
- 哪些内容进入待确认区
- 是否没有发现稳定规律
