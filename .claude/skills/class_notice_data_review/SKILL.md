# class_notice_data_review

## 功能

读取店铺数据采集结果、历史发布数据、已发布笔记记录，判断高成交笔记、高阅读低成交笔记、低阅读高转化笔记、低效方向，并输出下一轮班级通知笔记生成指令。

本 Skill 只做数据复盘和下一轮内容指令，不生成完整笔记，不做图，不自动发布。

## 触发语

- 复盘班级通知数据
- 生成下一轮笔记方向
- 运行今日数据复盘

## 执行前读取

1. `CLAUDE.md`
2. `00-系统协议与总规则/当前主推产品说明.md`
3. `02-产品资产库/01-教师产品/班级通知.md`
4. `09-创作规则库/班级通知内容生产总规则.md`
5. `16-复盘进化库/系统进化规则.md`

## 输入来源

优先读取：

- `F:\小红书店铺数据采集软件\采集结果`
- `F:\小红书店铺数据采集软件\笔记生产交接`
- `15-发布数据与店铺数据`
- `12-笔记生成结果/05-已发布`

如果某个来源不存在，记录到报告的“缺失数据来源”中，不要中断整个复盘。

## 输出位置

- `11-笔记生成任务/YYYY-MM-DD-班级通知-下一轮笔记生成指令.md`
- `15-发布数据与店铺数据/每日数据分析报告/YYYY-MM-DD-数据复盘.md`
- `16-复盘进化库/待确认复盘发现/YYYY-MM-DD-复盘发现.md`

如果输出目录不存在，先创建目录。

## 必须输出

每次复盘必须包含：

- 本次数据口径
- 一句话结论
- 高成交笔记
- 高阅读低转化笔记
- 低阅读高转化笔记
- 有效内容方向
- 低效内容方向
- 下一轮笔记生成指令
- 建议生成的版本
- 发布后观察指标

## 判断规则

### 高成交笔记

优先看订单数、成交金额、支付转化率、商品点击率。阅读不高但订单强，也应标记为值得放大。

### 高阅读低转化笔记

阅读或播放高，但商品点击率、订单、支付转化弱。通常说明封面吸引看热闹，但购买场景不够具体。

### 低阅读高转化笔记

阅读不高，但商品点击率或支付转化率高。通常说明方向值得复刻，需要加强封面和主图表达。

### 低效方向

重点识别泛表达：

- 教室通知神器
- 大屏可以呼叫学生
- 教师办公平台
- 家校沟通
- 家长通知
- 回执统计

## 班级通知优先复盘维度

- 不用跑教室叫学生
- 办公室发通知到教室一体机
- 叫班长拿试卷
- 叫课代表拿作业
- 叫组长收作业本
- 老师少跑腿
- 电子屏幕文案是否具体

## 下一轮指令格式

输出给 `class_notice_note_generator` 的指令应包含：

- 产品：班级通知
- 目标用户
- 主推痛点
- 推荐方向
- 推荐标题角度
- 推荐爆款结构
- 需要生成几套笔记
- 图片风格建议
- 主图电子屏幕要求
- 副图建议
- 正文风格
- 标签方向
- 本轮建议生成的版本
- 发布后观察指标

## 质量要求

- 不只看阅读量，要同时看商品点击率和支付转化率。
- 不把高播放低成交内容当作主复刻方向。
- 不把未发布、无数据的内容写成高效结论。
- 所有结论都要写清数据依据。
