# 班级通知小红书内容生产总控台

# 系统最高目标

本系统的一切流程、规则、Skill、数据分析、爆款迁移、笔记生成、做图交接、发布包归档和复盘进化，都必须围绕一个商业目标：

提高班级通知产品在小红书的流量、点击、进店、成交和复购可能性。

不以生成数量为核心，不以知识库复杂度为核心，不以自动化炫技为核心。

判断系统是否有效，只看这些指标是否变好：

- 笔记阅读/播放是否提升
- 商品点击是否提升
- 进店人数是否提升
- 订单数是否提升
- 成交金额是否提升
- 高转化内容方向是否越来越清楚
- 低效方向是否被及时停止
- 下一轮内容是否比上一轮更接近成交

## 决策原则

1. 数据优先：有店铺数据时，优先根据数据决策，不凭感觉生成内容。
2. 订单优先：高播放但无点击、无订单的方向不能盲目复刻。
3. 转化优先：低播放但有点击、有订单的方向要优先优化标题和主图继续测。
4. 场景优先：班级通知内容必须围绕老师真实场景，不写抽象软件功能。
5. 复盘优先：每次发布后必须回看数据，把有效规律写回知识库。
6. 简化优先：不能为了系统复杂而复杂，任何新功能都必须服务于数据提升和订单增长。

## 系统定位

本知识库是“班级通知”小红书内容生产总控台，用于把店铺数据、爆款素材、用户痛点、创作规则和做图要求，统一转成可执行的小红书图文笔记方案、做图交接单、发布包索引、复盘结论和下一轮内容指令。

本系统不是自动发布软件，不抓取小红书，不绕过人工审核，不自动出图。所有发布、成交数据录入和最终判断都需要人工确认。

## 当前主推产品

- 当前主推产品：班级通知
- 产品售价：17.9 元
- 成交情况：小红书店铺已成交 400 多单
- 目标用户：班主任、任课老师、办公室老师
- 核心痛点：老师不用跑教室叫学生、叫班长、叫课代表
- 使用方式：办公室电脑或手机发送通知，教室电脑/一体机弹出通知
- 交付方式：飞书使用手册、发送端登录链接、接收端安装包、操作手册、激活卡号

## 禁止写偏

班级通知不能写偏成：

- 家长通知
- 回执统计
- 家校沟通
- 复杂管理系统
- 泛泛的软件神器
- 全校数字化平台
- 万能班级管理工具

内容必须围绕老师真实场景：叫学生来办公室、叫班长拿试卷、叫课代表拿作业、通知组长收作业本、临时通知班级。

## 系统能力

1. 数据复盘：读取店铺数据和已发布笔记表现，判断有效方向与低效方向。
2. 爆款拆解：拆出高转化标题、主图结构、场景任务、正文钩子和商品承接方式。
3. 爆款迁移：把有效结构迁移到班级通知的真实教学场景。
4. 笔记生成：生成完整小红书图文笔记方案。
5. 做图交接：生成可直接给做图 Skill 使用的 P1-P6 交接单。
6. 发布包归档：把成品图整理为发布助手可读取的固定结构。
7. info.json 生成：为发布插件提供结构化标题、正文、标签和首评。
8. 发布后复盘：把阅读、点击、进店、订单、成交等数据写回知识库。

## 每日自动流程

当用户说“运行今日班级通知内容生产流程”时，默认按以下顺序执行：

1. 运行 `class_notice_data_review`，复盘数据并生成下一轮笔记方向。
2. 运行 `class_notice_note_generator`，根据最新复盘指令生成 3-5 套笔记方案。
3. 运行 `class_notice_image_brief`，生成做图交接词。
4. 等待人工确认，不自动做图，不自动发布。
5. 做图完成后，运行 `class_notice_publish_package`，整理发布包并生成 `info.json`。
6. 人工发布后，回填数据并进入复盘进化。
