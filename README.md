# 班级通知小红书内容增长系统

## 系统目标

本系统最终目标不是搭知识库，而是持续分析店铺数据、持续产出能带来流量和订单的小红书笔记，让班级通知产品的数据越来越好、订单越来越多。

## 当前主推产品

班级通知

## 核心痛点

老师不用跑教室叫学生、叫班长、叫课代表。

## 当前系统能力

- 店铺数据复盘
- 笔记方案生成
- 做图交接
- 发布包归档
- info.json 生成
- 人工修改学习
- 发布后复盘进化

## 当前发布助手规则

每个发布包必须包含：

- 主图.png
- 副图1.png
- 副图2.png
- 副图3.png
- 副图4.png
- 副图5.png
- 发布文案.md
- info.json

info.json 的 content 字段必须使用 `\n\n` 保留真实段落换行。

## GitHub 仓库用途

这个公开仓库用于让 ChatGPT / Codex / Claude Code 读取项目状态、系统规则、Skill、发布包索引和复盘摘要。

## 不同步内容

本仓库不保存 Cookie、Token、API Key、客户隐私、完整订单数据、激活卡号完整列表、network-response 原始文件、E 盘图片原图。
