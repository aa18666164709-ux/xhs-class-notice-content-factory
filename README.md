# 小红书爆款迁移内容工厂

这是一个 Obsidian 本地知识库驱动的小红书内容生产系统，用于沉淀产品资料、用户痛点、爆款案例、拆解结果、迁移规则、创作规则和图片规则，让 AI 能在读取本地资料后，一键生成完整的小红书图文笔记方案。

## 第一阶段范围

本阶段只创建本地知识库目录和 Markdown 模板文件。系统不做自动发布、不做小红书抓取、不接 MCP、不自动出图、不做复杂程序和数据复盘功能。

## 未来一套完整笔记方案必须包含

1. 标题
2. 正文
3. 标签
4. P1 主图文案
5. P1 主图怎么做
6. P2-P6 副图文案
7. P2-P6 副图怎么做
8. 评论区置顶文案
9. 私信承接话术
10. AI 自检结果

## 第一阶段使用方法

1. 把产品资料放入 `02-产品资产库/`，例如先维护 `期末评语生成助手.md`。
2. 把真实用户痛点、评论原话和使用场景放入 `03-用户需求与场景库/`。
3. 把人工收藏的爆款案例放入 `04-爆款案例库/`。
4. 按 `05-爆款拆解库/00-爆款拆解模板.md` 拆解爆款。
5. 按 `06-爆款迁移库/02-爆款迁移模板.md` 把爆款结构迁移到自己的产品。
6. 在 `11-笔记生成任务/04-任务模板.md` 新建任务，让 AI 读取本地资料并生成方案。

## 目录说明

- `00-系统协议与总规则/`：约束 AI 的执行边界、命名规则、禁止表达和自检规则。
- `01-输入素材池/`：临时收集截图、草稿、评论和灵感，等待整理入库。
- `02-产品资产库/`：沉淀产品卖点、使用场景、用户收益和转化承接方式。
- `03-用户需求与场景库/`：沉淀用户痛点、购买顾虑、原话和典型使用场景。
- `04-爆款案例库/`：保存待拆解、同行业和跨行业爆款案例。
- `05-爆款拆解库/`：拆出标题、钩子、结构、情绪和转化话术。
- `06-爆款迁移库/`：把爆款结构迁移到自己的产品和人群。
- `07-选题库/`：管理待验证、高效和低效选题。
- `08-关键词与搜索库/`：沉淀搜索词、标签词和高转化词。
- `09-创作规则库/`：规范标题、正文、标签、评论和私信话术。
- `10-图片方案库/`：规范 P1-P6 图文结构、排版、风格和提示词。
- `11-笔记生成任务/`：记录单条或批量笔记生成任务。
- `12-笔记生成结果/`：按检查、修改、发布和复盘状态存放输出方案。
- `13-内容流水线/`：跟踪内容从选题到复盘的人工流程。
- `14-人工修改学习库/`：保存 AI 原稿、人工修改稿和偏好规律。
- `15-发布数据与店铺数据/`：后续人工录入发布数据和店铺数据。
- `16-复盘进化库/`：后续人工沉淀有效结构、低效表达和优化建议。
- `17-模板库/`：集中存放可复制的通用模板。
- `18-运行日志/`：记录每次 AI 生成、修改和人工判断过程。
